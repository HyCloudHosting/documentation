# Console

The Console is one of the most important tools available in the Pterodactyl Control Panel.

It allows you to monitor your Minecraft server in real time, execute commands, and view everything happening while the server is running.

The console updates automatically and displays live information directly from your server.

---

## What You Can Do

Using the Console, you can:

- View server startup logs.
- Monitor player activity.
- Execute server commands.
- View plugin and mod messages.
- Identify server errors.
- Monitor crashes during startup.
- Check server warnings.

Almost every administrative task begins by checking the console.

---

## Sending Commands

You can execute any Minecraft server command directly from the command box located below the console.

Examples include:

```text
stop
```

Safely stops the server.

```text
list
```

Displays all connected players.

```text
say Server maintenance will begin in 5 minutes.
```

Sends a message to every player online.

```text
save-all
```

Forces the server to save all world data.

---

## Viewing Live Logs

While your server is running, the console continuously displays live output.

This includes:

- Server startup progress.
- Players joining and leaving.
- Plugin messages.
- Warnings.
- Errors.
- Server shutdown information.

Monitoring these logs is often the fastest way to identify problems.

---

## Console During Startup

When starting your server, you can monitor the entire boot process from the console.

If the server fails to start, the console will usually display the reason for the failure.

Examples include:

- Missing files
- Plugin errors
- Invalid configuration
- Unsupported Minecraft versions
- Java exceptions

If you're experiencing startup issues, always review the console before contacting support.

---

## Best Practices

For the best experience, we recommend:

- Read error messages completely before taking action.
- Stop the server properly using the `stop` command.
- Avoid repeatedly force-restarting the server.
- Keep a copy of important error messages when requesting support.

---

## Common Mistakes

Avoid the following:

- Closing your browser while a critical operation is running.
- Ignoring warnings displayed in the console.
- Executing commands copied from untrusted sources.
- Repeatedly restarting a server that is crashing.

---

## Need Help?

If you don't understand an error shown in the console, create a support ticket through our Discord server and include:

- A screenshot of the console.
- The complete error message.
- The approximate time the issue occurred.

Providing this information helps our team diagnose the issue much faster.
