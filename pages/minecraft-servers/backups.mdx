# Backups

Backups allow you to save a copy of your Minecraft server so you can restore it later if something goes wrong.

Whether you're installing plugins, updating your server, or making major configuration changes, creating a backup beforehand is highly recommended.

---

## Why Create Backups?

Creating regular backups helps protect your server from:

- Accidental file deletion.
- Corrupted worlds.
- Failed plugin installations.
- Configuration mistakes.
- Server software changes.
- Unexpected issues during updates.

A recent backup can save hours of work.

---

## Creating a Backup

To create a new backup:

1. Log in to the Control Panel.
2. Open your Minecraft server.
3. Navigate to **Backups**.
4. Click **Create Backup**.
5. Enter a name (optional).
6. Confirm the backup creation.

The backup process will begin immediately.

Depending on the size of your server, this may take several minutes.

---

## Restoring a Backup

If you need to restore your server:

1. Open the **Backups** page.
2. Locate the backup you want to restore.
3. Click **Restore**.
4. Confirm the operation.

> **Warning**
>
> Restoring a backup replaces your current server files with the contents of the selected backup.
>
> Any changes made after the backup was created will be permanently lost.

Always verify that you have selected the correct backup before restoring it.

---

## Downloading Backups

You can download backups to your local computer for additional protection.

Keeping an offline copy is recommended for important servers.

---

## Deleting Backups

Unused backups can be deleted from the control panel to free available backup slots.

Before deleting a backup, make sure it is no longer needed.

Deleted backups cannot be recovered.

---

## When Should You Create a Backup?

We recommend creating a backup before:

- Installing plugins.
- Installing modpacks.
- Updating Minecraft.
- Changing server software.
- Editing important configuration files.
- Uploading a new world.
- Performing a server reinstall.

Making backups a habit greatly reduces the risk of permanent data loss.

---

## Best Practices

To help keep your server safe:

- Create backups regularly.
- Keep multiple restore points.
- Download important backups to your computer.
- Name your backups descriptively.
- Verify successful backup creation before making major changes.

---

## Backup Limitations

The number of backups available depends on your hosting plan.

If you've reached your backup limit, you'll need to delete an existing backup before creating a new one.

You can always view your current backup usage from the control panel.

---

## Need Help?

If a backup fails to create or restore correctly, please contact our support team through Discord or the Client Area.

When requesting assistance, include:

- The backup name (if applicable).
- Any error message displayed.
- The approximate time the issue occurred.
