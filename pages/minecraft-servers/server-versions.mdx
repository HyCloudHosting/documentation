# Server Versions

Minecraft is updated regularly, introducing new features, gameplay changes, and bug fixes.

HyCloud Hosting allows you to run different Minecraft server versions depending on the software you selected when ordering your server.

Changing your server version is usually a simple process, but it should always be done carefully to avoid compatibility issues.

---

## Before Changing Versions

Before updating or downgrading your server, we strongly recommend:

- Creating a full backup.
- Verifying plugin compatibility.
- Verifying mod compatibility (if applicable).
- Reading the changelog of the new version.
- Testing major updates before using them on a production server.

A backup allows you to restore your server if something goes wrong during the update.

---

## Updating Your Server Version

Most server software allows you to change the Minecraft version by modifying the server version settings or replacing the server software.

The exact process depends on the software you're using.

After updating:

1. Start the server.
2. Watch the console for errors.
3. Verify that plugins or mods load correctly.
4. Join the server to test gameplay.

---

## Downgrading a Server

Downgrading to an older Minecraft version is possible in some situations, but it is generally **not recommended**.

Older versions may not support:

- New world generation.
- New blocks and items.
- Updated player data.
- Recently released plugins.

Attempting to downgrade a world may result in corruption or data loss.

---

## Plugin Compatibility

Plugins are built for specific Minecraft versions.

Before updating your server, always verify that your plugins support the version you plan to use.

Using outdated plugins may result in:

- Startup errors.
- Missing features.
- Performance issues.
- Server crashes.

---

## Mod Compatibility

If you're using Forge or another modded server software, every installed mod must support your selected Minecraft version.

Mixing incompatible versions may prevent the server from starting.

Always download mods that match both:

- Your Minecraft version.
- Your server software version.

---

## World Compatibility

Minecraft worlds are generally forward-compatible, meaning newer versions can usually load worlds created in older versions.

However, once a world has been opened in a newer version, it may no longer work correctly in older versions.

For this reason, always create a backup before opening a world in a newer Minecraft version.

---

## Recommended Update Process

For the safest experience:

1. Create a backup.
2. Verify plugin and mod compatibility.
3. Update your server version.
4. Start the server.
5. Review the console for errors.
6. Test all important features.
7. Allow players to join.

Following this process greatly reduces the risk of data loss or compatibility problems.

---

## Best Practices

We recommend:

- Keep backups before every update.
- Update plugins after updating Minecraft.
- Read plugin release notes.
- Avoid updating immediately before important events.
- Test major version changes whenever possible.

---

## Common Mistakes

Avoid the following:

- Updating without creating a backup.
- Mixing incompatible plugins or mods.
- Downgrading worlds without testing.
- Ignoring console warnings after updating.
- Assuming every plugin supports the latest Minecraft release.
