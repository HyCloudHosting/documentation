# Installing Modpacks

Modpacks completely transform the Minecraft experience by combining multiple mods into a single, pre-configured package.

Whether you're hosting a small private world or a large modded community, HyCloud Hosting allows you to install modpacks manually using the File Manager or SFTP.

---

## Before You Begin

Before installing a modpack, make sure that:

- Your server is running compatible server software (such as Forge or NeoForge).
- The modpack supports your Minecraft version.
- You have created a recent backup.
- Your hosting plan has enough RAM for the selected modpack.

Large modpacks often require significantly more resources than vanilla Minecraft.

---

## Downloading a Modpack

Always download modpacks from trusted sources.

Before downloading, verify:

- The Minecraft version.
- The required server software.
- Installation instructions provided by the modpack author.

Some modpacks provide separate downloads for the client and the server.

Always download the **server version** when hosting a server.

---

## Installing a Java Edition Modpack

To install a Java Edition modpack:

1. Stop your server.
2. Create a backup.
3. Remove the existing server files if required by the modpack documentation.
4. Upload the server files provided by the modpack.
5. Verify that the server software matches the modpack requirements.
6. Start your server.
7. Monitor the console for errors.

The first startup may take several minutes while the server generates configuration files.

---

## Installing a Bedrock Edition Add-on or Pack

Bedrock Edition does not support Java modpacks.

Instead, Bedrock servers commonly use:

- Behavior Packs
- Resource Packs
- Add-ons

The installation process depends on the server software you're using.

Always follow the instructions provided by the pack's developer.

---

## Updating a Modpack

Before updating:

1. Create a backup.
2. Read the modpack's release notes.
3. Replace or update the required files.
4. Start the server.
5. Verify that all mods load correctly.

Never update a modpack without first creating a backup.

---

## Removing a Modpack

To remove a modpack:

1. Stop the server.
2. Create a backup if you wish to preserve your world.
3. Remove the modpack files.
4. Upload a different server installation if desired.
5. Start the server again.

Some worlds may not function correctly after removing mods.

---

## Current Limitations

At this time, HyCloud Hosting does **not** provide a one-click Modpack Installer through the control panel.

All modpacks must be installed manually using the File Manager or SFTP.

A built-in modpack installation feature may be introduced in the future.

---

## Best Practices

We recommend:

- Always create a backup before installing or updating a modpack.
- Read the official installation guide.
- Verify compatibility with your Minecraft version.
- Allocate enough RAM for large modpacks.
- Install only one modpack at a time.

---

## Common Mistakes

Avoid the following:

- Installing the client version instead of the server version.
- Mixing mods from different Minecraft versions.
- Updating without creating a backup.
- Ignoring dependency requirements.
- Using incompatible server software.
