# Backups

Backups allow you to create snapshots of your Hytale server, making it possible to restore your files if something goes wrong.

Regular backups help protect your server against accidental file deletion, configuration mistakes, corruption, or other unexpected issues.

---

## Why Backups Are Important

We strongly recommend creating a backup before making significant changes to your server.

This includes:

- Editing configuration files.
- Uploading new server files.
- Updating your server.
- Replacing existing files.
- Testing new features.

Having a recent backup allows you to quickly restore your server if necessary.

---

## Creating a Backup

To create a backup:

1. Log in to the Control Panel.
2. Select your Hytale server.
3. Open the **Backups** page.
4. Click **Create Backup**.
5. Enter an optional name for the backup.
6. Wait for the backup process to complete.

The time required depends on the size of your server.

---

## Restoring a Backup

To restore an existing backup:

1. Open the **Backups** page.
2. Locate the backup you want to restore.
3. Click **Restore**.
4. Confirm the operation.

Your server files will be replaced with the contents of the selected backup.

---

> **Warning**
>
> Restoring a backup will overwrite your current server files. Any changes made after the backup was created may be permanently lost.

---

## Downloading Backups

If downloading backups is available for your server, you can save a local copy to your computer for additional protection.

Keeping an offline copy provides an extra layer of security in case you need to restore your server in the future.

---

## Deleting Backups

Old backups that are no longer needed can be deleted from the **Backups** page.

Deleting unnecessary backups helps keep your backup list organized and ensures you can easily identify your most recent restore points.

---

## Best Practices

We recommend:

- Create a backup before making major changes.
- Keep multiple recent backups instead of relying on a single one.
- Give backups descriptive names.
- Verify that important backups complete successfully.
- Download important backups for long-term storage when available.

---

## Common Mistakes

Avoid the following:

- Updating your server without creating a backup.
- Restoring an outdated backup by mistake.
- Deleting your only recent backup.
- Assuming backups are created automatically.
